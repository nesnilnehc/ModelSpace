# ModelSpace 3D 可视化优化计划

基于当前实现和 MCP 截图验证，制定可执行的优化清单。按优先级与依赖关系排序。

---

## 1. 高优先级：推广图导出质量

### 1.1 导出时轴标签可读性
- **现状**：轴刻度标签使用 `axisLabelOpts`，未针对导出放大
- **优化**：导出前临时提升轴标签 `scaleFactor`，或为导出模式使用更大字号
- **文件**：`src/app.js` `buildAxis`、`exportCanvasImage`

### 1.2 导出裁剪边界微调
- **现状**：`getCognitiveSpaceBoundsInPixels({ tight: true })` 使用 `margin: 6`
- **优化**：根据实际导出效果调整 `margin`，避免轴线或端点被裁切
- **文件**：`src/app.js` `getCognitiveSpaceBoundsInPixels`

### 1.3 导出专用视角预设（可选）
- **现状**：用户手动调节视角后导出
- **优化**：新增「推广图视角」按钮，一键切换到适合 README 的固定角度，再导出
- **文件**：`src/app.js`、`cognitive-model-3d.html`、`src/app3d/i18n.js`

---

## 2. 中优先级：视觉与交互

### 2.1 节点标签重叠
- **现状**：紧凑标签 `scaleFactor: 0.02`，多模型同格时可能重叠
- **优化**：按距离相机排序标签可见性，或进一步缩小 scaleFactor；或限制同时显示标签数量
- **文件**：`src/app.js` `updateLabelVisibility` 逻辑

### 2.2 坐标轴与背景对比
- **现状**：轴线 `emissiveIntensity: 0.68`，端点 `0.72`，纯色背景 `#0a1220`
- **优化**：若导出图轴线不够明显，可微调 `emissiveIntensity` 或轴材质的 `color`/`emissive`

### 2.3 雾效与深色节点
- **现状**：`Fog(0x0a1220, 140, 300)`，远处节点可能偏暗
- **优化**：可视情况调整 `near`/`far` 或节点 `emissiveIntensity`，保证远处节点仍可辨识

---

## 3. 低优先级：结构与维护

### 3.1 常量集中管理
- **现状**：`1.28`、`0.1`、`0.5`、`0.42` 等散落在 `buildNodes`、`buildAxis` 中
- **优化**：提取为 `VISUAL_CONFIG` 对象，便于统一调整和文档化
- **文件**：`src/app.js` 顶部

### 3.2 星场构建优化
- **现状**：`buildStarField()` 构建星场，`starFieldGroup.visible = false` 永不显示
- **优化**：若确认不再使用星场，可移除 `buildStarField` 调用及星场相关逻辑，减少初始化开销

### 3.3 导出流程文档化
- **现状**：README 已有简短说明
- **优化**：在 `docs/architecture/` 增加「推广图更新流程」说明：视角预设、筛选建议、导出步骤、保存路径

---

## 4. 执行顺序建议

| 阶段 | 任务 | 预估工作量 |
|------|------|------------|
| 1 | 导出裁剪边界微调（1.2） | 小 |
| 2 | 轴标签导出可读性（1.1） | 小 |
| 3 | 常量集中（3.1） | 小 |
| 4 | 推广图视角预设（1.3） | 中 |
| 5 | 节点标签重叠（2.1） | 中 |
| 6 | 星场移除（3.2） | 小 |
| 7 | 导出流程文档（3.3） | 小 |

---

## 5. 验收标准

- 导出 PNG 可直接作为 `docs/assets/modelspace-promo.png` 使用
- 三轴、刻度、节点在导出图中清晰可辨
- 透明背景导出正确，无 UI 元素残留
- 常量调整后视觉观感与当前一致或更优
